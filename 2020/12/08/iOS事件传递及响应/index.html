iOS事件传递及响应

Posted on 2020-12-08

有这样一种场景，多张图片的展示一般会使用类似九宫格的布局，删除的按钮一般是放在图片的右上角一个x的按钮，点击即删除该张图片，然而这个按钮有一部分是在图片视图外的，我们会发现，点击视图外的部分是无法响应事件的，从而导致点击响应区域很小，点击删除体验不佳，为什么出现这种情况呢，那就与iOS的事件传递及响应机制有关了。

1、一个触摸事件的传递及响应可以大致分为以下几个步骤：

  • 1、用户触摸屏幕，通过硬件产生一个touch，包含待执行的事件及触摸的坐标等信息。
  • 2、从最底层UIApplication沿着UIApplication -> UIWindow -> UIVIewController.view -> subViews -> subViews …这条链路，通过- (UIView *)hitTest:(CGPoint)point withEvent:(UIEvent *)event方法查找出最终用户触摸的是哪一个view或者丢弃。
  • 3、查找到最终触摸的view进行事件响应，如果不能响应就沿着上面相反的链路进行传递，view -> superView -> … -> UIViewController -> UIWindow -> UIApplication -> AppDelegate 或者 丢弃。链路的下一级都是上一级的nextResponder.

2、寻找hitTestView的过程：

  • 1、从视图最底层UIWindow开始遍历它的subView（调用hitTest: withEvent:方法）；
  • 2、同一superview的subview，按照视图add的逆顺序遍历，即subviews的逆序遍历；
  • 3、找到hitTestView，返回hitTestView，查找结束。
  • 4、返回hitTestView过程：
    • 如果view的userInteractionEnabled = NO，hidden = YES，alpha < 0.05，enable = NO，返回nil；
    • 如果触摸坐标不在视图内，返回nil；
    • 逆序遍历子视图，调用hitTest: withEvent:获得hitTestView，不为空返回hitTestView，若为空返回自身视图做为hitTestView。

如果父视图userInteractionEnabled = NO，其子视图点击事件则无法响应。

找到 hitTestView 之后，事件就交给它来处理，hitTestView 就是 firstResponder（第一响应者），如果它无法响应事件（不处理事件），则把事件交给它的 nextResponder（下一个响应者），即1中的步骤3，响应链路。

UIControl->UIView->UIResponder->NSObject这个继承关系。所有响应者的基类都是 UIResponder，UIApplication/UIView/UIViewController 都是 UIResponder 的子类
