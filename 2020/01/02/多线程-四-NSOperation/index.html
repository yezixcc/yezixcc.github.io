多线程(四)-NSOperation

Posted on 2020-01-02
  • 简介
  • NSOperation
    • NSInvocationOperation：
    • NSBlockOperation：
  • NSOperationQueue

NSOperation也是iOS中比较常见的多线程实现方式，GCD的基础上面向对象的一层封装，更加的面向对象，使用起来也很方便，也是任务和队列的概念。

简介
  • 作用
    • 配合使用NSOperation（任务）和NSOperationQueue（队列）实现多线程编程。
  • 实现多线程具体步骤
    • 将需要执行的任务封装到NSOperation对象中；
    • 将NSOperation对象添加到NSOperationQueue中；
    • 系统自动将NSOperationQueue中的NSOperation取出来；
    • 将取出的NSOperation对象中的任务放到一条新的线程中执行。
NSOperation

  • 是一个抽象类，不能直接使用，需借助其子类封装任务；

  • 使用NSOperation子类三种方式：

    • NSInvocationOperation；
    • NSBlockOperation；
    • 自定义子类继承NSOperation，实现内部对应方法。

  • 基本使用：

    NSInvocationOperation：

    打印结果：
    ——<NSThread: 0x600002f34680>{number = 1, name = main}

    NSInvocationOperation单独这样使用和performSelector没有区别，在主线程中执行，达不到多线程效果。所以单独使用没有多大意义，它就相当于包装任务，要和队列结合起来使用才有意义。

    NSBlockOperation：

    打印结果:
    4——<NSThread: 0x6000032da240>{number = 1, name = main}
    1——<NSThread: 0x60000329a540>{number = 4, name = (null)}
    2——<NSThread: 0x6000032836c0>{number = 5, name = (null)}
    3——<NSThread: 0x6000032ad640>{number = 6, name = (null)}

    可以看出这个达到了多线程的效果，任务1始终在主线程中执行。

NSOperationQueue

  • 主队列

    • [NSOperationQueue mainQueue]；
    • 任务(NSOperation)都在主线程中执行。

  • 非主队列

    • [[NSOperationQueue alloc] init];
    • 包含串行和并行功能；
    • 任务(NSOperation)都在子线程中执行，不管是串行还是并行；
    • 任务添加到队列即执行，不用调用start方法。

  • 基本使用：

    1.先创建operation，再加入队列。

    打印结果：
    op1 ———– <NSThread: 0x600003a74840>{number = 8, name = (null)}
    op2 ———– <NSThread: 0x600003a856c0>{number = 12, name = (null)}
    op3-1 ———– <NSThread: 0x600003a89ac0>{number = 13, name = (null)}
    op4 ———– <NSThread: 0x600003a74840>{number = 8, name = (null)}
    op3-2 ———– <NSThread: 0x600003a89f80>{number = 14, name = (null)}
    op3-3 ———– <NSThread: 0x600003a856c0>{number = 12, name = (null)}

    需要使用NSOperation的子类封装任务，可以看出任务添加到队列中后自动执行，不用调用start方法，自动的开启了子线程，默认情况下类似并行队列，那如何实现串行队列呢？

    为何还需要自定义NSOperation呢？当一个任务需要写很长的代码，并且很多地方经常用到，我们就可以封装一个NSOperation子类，用的时候只需要alloc init ,再加入到队列中就可以使用了。

    2.直接用block创建operation并加入队列。

    打印结果：
    ——–2 —- <NSThread: 0x6000008c4ac0>{number = 7, name = (null)}
    ——–1 —- <NSThread: 0x6000008e8b80>{number = 5, name = (null)}
    ——–0 —- <NSThread: 0x6000008eaa40>{number = 6, name = (null)}

    3.最大并发数。

    打印结果：
    2 ———– <NSThread: 0x600002a6a6c0>{number = 7, name = (null)}
    1 ———– <NSThread: 0x600002a633c0>{number = 8, name = (null)}
    3 ———– <NSThread: 0x600002a50380>{number = 3, name = (null)}
    4 ———– <NSThread: 0x600002a6a6c0>{number = 7, name = (null)}
    5 ———– <NSThread: 0x600002a50380>{number = 3, name = (null)}

    maxConcurrentOperationCount，设置最大并发数，同时只有maxConcurrentOperationCount个任务在执行。

    底层是GCD，所以执行任务是开启新线程还是重用旧线程有系统决定，程序员无法控制，设置最大并发数，在一定程度上控制了开启线程的数量，避免开启过多线程导致资源占用。

    maxConcurrentOperationCount = -1，默认值。

    maxConcurrentOperationCount = 0，不会执行任务。

    maxConcurrentOperationCount = 1，串行执行任务。

    maxConcurrentOperationCount > 1，并发执行任务。

    4.suspended。

    当用户操作界面时，我们希望界面更加流畅，而该页面有个队列在执行任务，就可以使用suspended，停止该队列中还未执行的任务，等操作完成后再继续执行队列中未执行的任务。

    5.cancelAllOperations。

    cancelAllOperations相当于调用队列中所有Operation的cancel方法，同样的，只能取消当前还未执行的任务，已经执行的不受控制会继续执行。

    注意的是，苹果官方建议，自定义的NSOperation，想要执行cancel方法，建议在main里面，执行完一段耗时操作后，自己判断isCancelled，如有取消，直接return，手动控制后面的任务不再执行。如果不做特殊处理，调用[self.queue cancelAllOperations]，对于自定义的NSOperation main中任务不会停止执行，达不到原有cancel的效果。

    6.addDependency。

    打印结果：
    op1 ——- <NSThread: 0x600001c45740>{number = 10, name = (null)}
    op4 ——- <NSThread: 0x600001c45480>{number = 12, name = (null)}
    op2 ——- <NSThread: 0x600001c59240>{number = 13, name = (null)}
    op3 ——- <NSThread: 0x600001c59240>{number = 13, name = (null)}

    可以看出，op3任务的执行在op1，op2，op4之后。但切记不要互相依赖，上面是同一个队列的任务之间的依赖，也可以是不同队列中任务间添加依赖。

    7.completionBlock。

    打印结果：
    op1-2 ——- <NSThread: 0x600000f030c0>{number = 3, name = (null)}
    op1-3 ——- <NSThread: 0x600000f0a480>{number = 5, name = (null)}
    op1-1 ——- <NSThread: 0x600000f057c0>{number = 4, name = (null)}
    op1完成 ——- <NSThread: 0x600000f0a480>{number = 5, name = (null)}

    利用completionBlock监听操作中的任务执行完成，并且操作完成后的block中还是子线程。

    8.线程间的通信。

    1、子线程下载图片，主线程显示图片。

    2、子线程下载并合成图片，主线程显示图片。

    线程间通信主要就是子线程处理耗时操作，完成之后再回到主线程刷新UI。上面image1为什么要用__block呢？image1是一个局部变量，方法执行完成即会销毁，然后此时block可能还没有回调，等回调回来block中使用image1时它已经销毁了，所以加上避免销毁。
