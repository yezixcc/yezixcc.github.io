线性结构

Posted on 2020-06-15

线性结构是一种数据结构，它是由若干个数据元素按照顺序组成的结构，其中每个数据元素都有唯一的前驱和后继（除了第一个元素没有前驱，最后一个元素没有后继）。在线性结构中，数据元素之间存在一对一的关系。

一、数组

1、简介

数组是一种顺序存储的线性表，所有元素的内存地址是连续的。

在很多编程语言中，数组都有个缺点：无法动态修改容量，即创建的时候指定多少容量就是多少，后续无法再增加，但在开发中更希望数组的容量可以动态改变，就像oc的可变数组，实际底层就是苹果实现了对数组的封装，设计了一个可变数组供开发者使用。

实现动态数组的几个要点：

  • 自动扩容（在添加元素时，检查是否达到了静态数组的容量上限。如果达到了，则将数组扩容为原来的两倍，并把旧数据复制到新数组中。）
  • 静态数组
  • 元素计数器size
2、复杂度

  • 查：E get(int index)，O(1)；

  • 改：E set(int index, E element)，O(1)；

  • 增： add(int index, E element)

    • 最好情况复杂度：往最后一个位置添加元素，不需要挪动元素。即O(1)；
    • 最坏情况复杂度：往第一个位置添加元素，需要挪动全部元素。即O(n)；
    • 平均情况复杂度：往中间任一位置添加元素，取最好和最坏的平均，即(1+2+..+n)/n，即O(n/2)，即O(n)。

  • 增： add(E element):

    • 最好情况复杂度：不需要扩容的情况下，O(1);
    • 最坏情况复杂度：需要扩容，则需要把原有的数组所有元素赋值到新扩容的数组，故O(n)；
    • 平均情况复杂度：只要最后一次达到数组容量的时候复杂度才是O(n)，平均下来还是O(1);
    • 均摊情况复杂度：像数组这种请添加元素的情况，只有个别操作复杂度猛增，所以引申出这种复杂度，把最后一次均摊到前面次数的每一次中，故O(1)。一般来说，均摊复杂度=最好复杂度。

  • 删： E remove(int index)：

    • 最好情况复杂度：删除最后一个元素，无需挪动元素。即O(1)；
    • 最坏情况复杂度：删除第一个元素，需要挪动全部元素。即O(n)；
    • 平均情况复杂度：删除中间任一位置元素，和添加一样，即O(n)。
3、扩容

如果内存紧张，动态数组有比较多的剩余空间，可以考虑对动态数组缩容，释放掉未使用的内存空间，和扩容一样，都是申请一块新的内存空间，只不过这个内存空间比原来的小很多，然后把数组元素赋值到新的内存空间中。缩容肯定是在remove操作中进行，还有clear中，可以自行定义一个规则，size和数组容量进行比较，达到一定的条件后就进行缩容操作，扩容倍数和缩容倍数相乘不等于1就好，否则可能引起复杂度震荡。

小知识点：

  1. 位运算的效率最高

    位运算的效率是最高的，左移1 << 1，相当于乘以2，右移1 >> 1，相当于除以2.

    如：4 << 1 = 8; 4 >> 1 = 2;

  2. 对象数组 vs 整型数组

    整型数组：直接指向存放数据的内存地址。

    对象数组：存放的是指向对象内存的指针地址，存的是指针地址，并不是实际的元素占用的内存。

  3. Java中，int的对象类型：Integer，double 对象类型：Double， 默认重写了equals方法，比较的是值，并不是内存地址。equals方法不重写默认是比较内存地址。

4、优化

往最前面和最后面插入删除元素不挪动元素；往中间插入删除元素最多挪动size/2次；

定义一个int变量first，用来记录数组首元素的位置，一开始first = 0，若删除首元素的时候，无需挪动元素，只需要把first = 1，清空原首元素内容就好，遍历数组的时候就从索引1开始了。

二、链表

动态数组有个明显的缺点：可能造成内存空间的浪费，能否用到多少内存就申请多少内存？这就需要用到链表结构了，用到时再开辟内存。

1、单向链表

由若干个节点（Node）按照线性顺序组成，所有元素的内存地址不一定是连续的。每个节点包含两部分：数据域和指针域（指向下一个节点）。

1、链表设计
2、clear

头结点指向null，就没有再引用后面的节点了，后面的就自动释放了。

3、add(int index, E element)
  1. 找到添加位置的前一个节点；
  2. 将新添加节点的next指向前一个节点的next；
  3. 将前一个节点的next指向添加的节点。
4、remove(int index)

获取到前面的节点(prev)，prev.next指向prev.next.next。

5、int indexOf(E element)

6、复杂度

  • 查 E get(int index)：

    • 最好情况复杂度：获取首节点。即O(1)；
    • 最坏情况复杂度：获取最后一个节点。即O(n)；
    • 平均情况复杂度：查找中间节点，取最好和最坏的平均，即(1+2+..+n)/n，即O(n/2)，即O(n)。

  • 改 E set(int index, E element)：

    修改节点需要先查找到对应的节点才能进行修改，所以复杂度和查一样。

  • 增 add(int index, E element)：

    增加一个节点，需要首先找到index前面的那个节点，所以复杂度和查一样。

  • 删 E remove(int index)：

    删除一个节点，也需要首先找到index前面的那个节点，所以复杂度和查一样。

注意：有些资料说链表的优势是添加和删除的复杂度是O(1)，其实是指添加，删除的那一刻复杂度是O(1)，具体就是指改变next的指向这个操作，链表的添加和删除操作只需要改变next的指向就好，不像数组需要一个个挪动元素，而实际上完成整个插入删除操作的复杂度是O(n)。

2、双向链表

单向链表只有首节点，查找某个节点只能从首节点挨个next去查找，如果要查找最后一个节点也只能从首节点开始，性能低，所以就出现了双向链表，如果是查找比较靠后的节点，就可以从尾节点开始找，综合性能得到提升。

双向列表就是在单向链表的基础上增加了一个尾节点指针，指向上一个节点，这样进行查找的时候，既可以从首节点开始，也可以从尾节点开始，具体依据index而定，从而提升链表综合性能。

2、双向链表 vs 单向链表
  • 单向链表：1+2+3…+n，取平均 1/2+n/2;
  • 双向链表：(1+2+3…+n/2)*2，因为是从首尾都可以查，所以最多加到n/2，取平均1/2+n/4；

由此看出双向链表比单向链表几乎节省一半的操作数量。

6、双向链表 vs 动态数组
  • 动态数组：开辟，销毁内存空间次数相对较少，但可能造成内存空间浪费（因为有时需要扩容），但可以利用缩容来解决；
  • 双向链表：开辟，销毁内存空间次数相对较多，但不会造成内存空间的浪费。

如果频繁在尾部进行添加，删除操作，动态数组，双向链表均可选择；

如果频繁在头部进行添加，删除操作，建议使用双向链表；

如果有频繁的在任意位置进行添加、删除操作，建议使用双向链表；

如果有频繁的查询操作（随机访问），建议使用动态数组；

有了双向链表，单向链表是否就没有任何用处了？并不是，在哈希表的设计中就用到了单链表。

4、静态链表

之前接触的链表都有首结点，也就是一个first指针，如果没有指针的情况下如何实现链表呢？有些语言确实是没有链表的。可以用数组来模拟链表，这就是静态链表。

其他链表都是通过指针实现的，静态链表使用的是数组来模拟链表；

  • 使用一个数组：数组每个元素存储2个数据，一个放值，一个放下个元素的索引；
  • 使用两个数组：数组1存值，数组2存索引；

平时开发中没有必要使用静态链表。

三、栈

栈是一种特殊的线性表，只能在一端进行操作。先进后出原则，只能移除栈顶元素。

主要有两个操作：添加元素push，删除元素pop。这个栈和栈空间不是同一个概念，这个是一种数据结构，栈空间是指一种内存空间。

可以用以下两种数据结构来实现：

  • 动态数组

  • 链表

1、栈的实现

可以利用数组来实现，实际就是操作数组的最后一个元素，push就相当于add(size-1, element)，pop就相当于remove(size-1)。

2、栈的应用

  1. 浏览器的前进和后退

    输入jd.com，输入qq.com，输入baidu.com;
    后退 回到qq.com;
    再后 退回到jd.com;
    前进 进入qq.com;
    输入 taobao.com;
    此时无法前进了 — 说明baidu.com已经删除了
    后退 回到qq.com。

    利用两个栈可以实现以上功能，A栈存输入的网址，B栈存后退出的网址，输入新的网址时清空B栈数据。

  2. 软件的撤销与恢复功能

四、队列

1、普通队列

队列是一种特殊的线性表，只能在头尾两端进行操作。先进先出原则，只能从队尾添加元素，只能从队头删除元素。

可以用以下来实现：

  • 双向链表
  • 动态数组

利用双向链表来实现队列的功能，因为队列主要是往头和尾操作元素，正好双向链表有首结点和尾结点，操作起来复杂度都是O(1)。

设计一个队列，里面包含一个双向链表，利用双向链表来存放所有元素。利用这种组合的设计方式，而不是继承自一个双向链表，如果采用继承的方式，使用队列时，双向链表的一些方法也能被调用，这违背队列的特性，所以采用组合的方式更合理。

  • 入队（enQueue）：即链表的尾部添加元素。list.add(element)；

  • 出队（deQueue）：即删除链表的头部元素。list.remove(0)；

2、双端队列

普通队列只能从队尾添加元素，队头删除元素，双端队列能在头尾两端添加，删除元素。比普通队列多了两个方法。队尾出队和队头入队。

可以被用来解决某些限制条件下的问题，例如滑动窗口问题和求最大值最小值等等。

  • 队尾入队（enQueueRear）：即链表的尾部添加元素。list.add(element)；

  • 队头出队（deQueueFront）：即删除链表的头部元素。list.remove(0)；

  • 队头入队（enQueueFront）：即链表头部添加元素。list.add(0, element);

  • 队尾出队（deQueueRear）：即删除链表尾部元素。list.remove(list.size()-1)。

3、循环队列

把顺序队列首尾相连，把存储队列元素的表从逻辑上看成一个环，成为循环队列。当队头指针或者队尾指针到达数组的末尾时，会“循环”回到数组的开头继续存储数据。可以动态数组实现。

  • front 指向队首元素；
  • rear 指向队尾元素的下一个位置；
1、入队

将新元素插入到rear所指向的位置，并将rear后移一位，尾部空位不够将继续添加到头部的空位，当size大于我们设定的长度时，才进行动态扩容。

如果不进行扩容，当队列满时，将无法进行入队操作，因为此时队列已经满员。因此，对于任意大小的循环队列，最多只能存储 n-1 个元素，其中n为队列的容量，因为要留一个rear指向的空地址。

2、出队

删除front所指向的元素，并将front后移一位。由于队列是循环的，所有的操作都是在数组上进行的，不需要进行元素的移动，因此时间复杂度为常数级别，即O(1)。

3、设计

  • 容量：size-1;

  • 空队列：front == rear；

  • 数组真实索引：(front + index) % size；

  • 队列已满：(rear+1)%size == front，因为front不一定从0开始，所以取模运算，宗旨就是rear和front重合了。其中(rear+1)%size即为rear要指向的位置。

4、循环双端队列

和循环队列对比，多了从队头入队，队尾出队。

优化：尽量避免使 *，/，%，浮点数运算，效率低下。

已知n > =0，m>0，且n < 2m，n%m等价于n - (m > n ? 0 : m)。

五、哈希表

也叫“散列表”，数组索引作为key，元素内容作为value，就是一个哈希表。典型的空间换时间。

  • 通常使用数组来实现；
  • 每个关键字通过哈希函数对应数组的索引；
  • 插入、查找、删除等操作都可以在O(1)的时间复杂度内完成。
  • 每个数组元素叫做Bucket（桶）。
1、哈希冲突

2个不同的关键字，经过哈希函数计算到了得到了相同的结果（即计算出的数组索引值一样）。

2、解决哈希冲突
  • 开放地址法：按照一定的规则向其他地址探测，直到找到空桶为止；
  • 再哈希法：设计多个哈希函数；
  • 链地址法：通过链表把相同的index的元素串起来（Jdk 1.8中如果重复的元素数量大于8，会将链表转成红黑树，红黑树相当于排序数据）；
3、哈希函数

  1. 先生成key的哈希值；

  2. 再让key的哈希值跟数组的大小进行相关运算，生成一个索引值；

如何生成哈希值？

在java中，HashMap的key必须是实现hashCode，equals方法，也允许key为null；

  1. 整数：

    整数值当做哈希值，比如10的哈希值就是10，java中的哈希值必须是int类型。

  2. 浮点数：

    将存储的二进制格式转化为整数值。

  3. 字符串：

    jack由4个字符组成（字符的本质就是一个整数），jack的哈希值：

    j * n^3 + a * n^2 + c * n^1 + k * n^0 = [(j * n + a) * n + c] * n + k，在JVM中，n=31，31是一个奇素数，JVM会将31 * i优化成 (i << 5)-i，位运算性能要高于乘，除，取模。

  4. 自定义对象：

    需要自己实现hashCode和equals方法，把每个属性的哈希值按照 * 31的方式加起来。

    hashCode在计算索引时调用，equals在比较元素相等时调用（哈希冲突时。比较两个key收否相等），默认的hashCode比较是跟内存地址关联的，默认的equals是比较两个内存地址。
