3、环形链表

给定一个链表，判断链表中是否有环。

为了表示给定链表中的环，我们使用整数 pos 来表示链表尾连接到链表中的位置（索引从 0 开始）。 如果 pos 是 -1，则在该链表中没有环。

快慢指针思想：

定义两个指针，slow指针一次走一步，fast指针一次走两步，如果有环，那么slow和fast指针到最后一定会重合，即指向同一个节点；如果没有环，那么fast指针一定先达到NULL，结束查找。

虚拟头节点思想：

有时候为了让代码更加精简，统一所有节点的处理逻辑，可以在最前面增加一个虚拟的头结点（不存储数据）。

4、约瑟夫问题

8个人，每个人都有一个序号，从第1个开始，每次数3次，数到谁，谁被踢出，继续往下进行，到最后剩下的那个人胜利。

利用循环链表就可以解决，单向循环和双向循环都可以，在循环链表中新增以下几个方法。

  • 新增一个current指针，指向某个节点，走到哪指到哪；
  • 新增一个reset()方法，让current指向头节点；
  • 新增一个E next()方法，每执行一次就让current指向current的next，即往后走一步；
  • 新增一个E remove(E node)方法，删除指定的节点；

有了以上的方法就可以解决以上问题了

5、删除单链表的倒数第N个节点

使用快慢指针思想。两个指针slow和fast来遍历链表，初始时二者都指向虚拟头结点dummy（即dummy.next=head）；接着，fast先向后移动n个节点，然后slow和fast同时向后移动，直到fast到达链表末尾。

此时，slow指向的节点就是要被删除的节点的前一个节点。因为slow.next就是要被删除的节点，所以将slow.next指向slow.next.next即可完成删除操作。最后返回dummy.next即为新链表的头结点。

注意，这里引入了一个虚拟头结点dummy，这是为了方便处理头结点的特殊情况。如果没有虚拟头结点，需要额外判断头结点是否需要被删除。

二、栈

1、有效括号检测

给定一个只包括 ‘(‘，’)’，’{‘，’}’，’[‘，’]’ 的字符串，判断字符串是否有效。

有效字符串需满足：

左括号必须用相同类型的右括号闭合。
左括号必须以正确的顺序闭合。
注意空字符串可被认为是有效字符串。

输入: “()”
输出: true

输入: “()[]{}”
输出: true

输入: “(]”
输出: false

输入: “([)]”
输出: false

输入: “{[]}”
输出: true

可以利用栈解决这个问题，核心思路：

遍历字符串，遇到左括号就放入栈1中，遇到右括号就拿出栈1顶的元素与之匹配，如果不匹配则不是返回不是有效字符，匹配则继续往下执行， 直到字符串遍历完毕，此时，如果栈1中没有元素了，说明全部匹配完毕，是有效括号。

三、二叉树

1、二叉树直径

在二叉树中，直径是指任意两个节点之间最长路径的长度（经过的节点数-1），即为左右子树高度和的最大值，可以使用递归或栈来实现。

2、排序二叉树的插入

如果插入相同的数，需要做什么保证树可以还原（即不允许丢弃掉相同的数）？

1、当前节点是否为空，如果为空，直接插入；

2、插入的值与当前节点的值相同，则插入当前节点的右子树中；

3、插入的值小于当前节点的值，递归插入到左子树中；

4、插入的值大于当前节点的值，递归插入到右子树中；

3、翻转二叉树

栈和递归的复杂度都是O(n)。但是如果使用递归，层级太深，可能引发栈溢出，因为每次调用函数都要开辟栈帧，栈空间是比较小的。

4、如何验证二叉树是否相等？

判断左右子节点是否相等

  1. 如果两个节点都为空，则相等；
  2. 如果两个节点一个为空，一个不为空，则不相等；
  3. 如果两个节点的值不相等，则不相等。

1、两数之和

给出一个整型数组 numbers 和一个目标值 target，请在数组中找出两个加起来等于目标值的数的下标，返回的下标按升序排列。（注：返回的数组下标从1开始算起，保证target一定可以由数组里面2个数字相加得到）

输入：[3,2,4],6

返回值：[2,3]

说明：因为 2+4=6 ，而 2的下标为2 ， 4的下标为3 ，又因为 下标2 < 下标3 ，所以返回[2,3]

复杂度O(n)，只需要一次遍历即可；

2、写一个二分查找，mid计算的时候溢出怎么做？

mid溢出是指在计算mid时使用的整型数据类型不够大，left+right超出了范围，比如32位有符号位，最大是2^31-1，超出了这个数，导致计算（left+right)/2错误。

可以采用mid = left+(right-left)/2。

3、分数到小数

给定两个整数，分别表示分数的分子 numerator 和分母 denominator，以字符串形式返回小数。

如果小数部分为循环小数，则将循环的部分括在括号内。
如果存在多个答案，只需返回任意一个 。
对于所有给定的输入，保证答案字符串的长度小于 10^4。

输入：numerator = 1, denominator = 2 输出：”0.5”

输入：numerator = 2, denominator = 3 输出：”0.(6)”

输入：numerator = 4, denominator = 333 输出：”0.(012)”

使用长除法和HashMap解题；

  1. 将分子进行除法运算，得到整数部分和余数。
  2. 将余数乘以10，再进行一次除法运算，得到新的整数部分和新的余数。
  3. 将新的余数乘以10，再进行一次除法运算，如此循环下去，直到余数为0或者出现重复的余数。
  4. 如果余数为0，则分数是有限小数；如果出现重复的余数，则分数是循环小数，重复的部分即为循环节。

5、判断一个数组是不是另一个数组的子数组

int[] A = {1, 2, 3, 4, 5, 6}; int[] B = {3, 4, 5}，B就是A的子数组。

双指针法。一个指针遍历A，一个指针遍历目标数组B；

1、循环结束条件，A遍历完，B遍历完。

2、A[i] == B[j]，则j++；如果不等于就需要回溯，j回到初始位置，i要保持当前的位置不能变。

3、i++。

4、最后j == B.length就算正确。

7、RGB排序

给定一个只有“R”“G”“B”三个字母的字符串，用一次遍历基于原字符串将他们以“RGB”的顺序排序好。

1、如果不考虑遍历次数，可以用计数排序法，统计R,G,B出现的次数，然后再一次取出拼接。

2、如果只能遍历1次，并且不能使用数组开辟其他空间，使用三指针法。类似快排思想。

1、left指针指向排好序的R的下一个位置，初始位置0；(从头向尾)；

2、right指针指向排好序的B的前一个位置，初始位置arr.length-1;（从尾向头）

3、cur指向当前遍历的位置，从头向尾，初始位置0；

循环遍历：

cur的值 == R，就和left指针值进行交换，left++，向后移一位；

cur的值 == B，就和right指针值进行交换，right–，向前移一位；

cur的值 == G，不交换，后移一位cur++;；

直到cur <= right结束循环，全部遍历完成。
