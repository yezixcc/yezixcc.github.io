复杂度

Posted on 2020-06-11

复杂度（Complexity）在计算机科学中是用来衡量算法、问题或系统的性能和效率的概念。

常见的复杂度有两种：时间复杂度和空间复杂度。

二、如何评判算法好坏？

  • 正确性，可读性，健壮性
  • 时间复杂度：估算程序指令执行的次数，即决定了执行时间的长短。
  • 空间复杂度：估算所需占用的内存空间（比如变量比较多等）。

第一要位的肯定是算法的正确性，要确保算法能够实现需求，其次是算法的时间复杂度，至于空间复杂度在现如今硬件越来越强大的条件显得不是特别紧要，所以在算法正确的情况下，对算法进行优化重点就在于时间复杂度的优化，不同的算法解决同样的问题，执行时间可能存在巨大的差距，如上求斐波拉契数的两个算法。

1、时间复杂度

如何来计算算法的时间复杂度？既然它是指程序指令执行的次数，那么就可以大概计算下算法中指令次数之和，下面计算下些算法的时间复杂度。

1、大O表示法

由上引申出大O表示法，用来描述时间复杂度，有以下几个原则。

  • 忽略常数
    • 5 => O(1) – 常数阶
  • 忽略系数
    • 2n + 5 => O(n) – 线性阶
  • 忽略低阶
    • 2n^2 + 2n + 5 => O(n^2) – 平方阶
    • 2n^3 + n^2 + 2n + 5 => O(n^3) – 立方阶
  • 对数阶一般省略底数
    • log2(n) = log2(5) * log5(n) ，统一叫log(n) => O(logn) – 对数阶
    • 2nlog2(n) + 2log2(n) + 1 => O(nlogn) – nlogn阶
  • 复杂度大小
    • O(1) < O(logn) < O(n) < O(nlogn) < O(n^2) < O(n^3) < O(2^n) < O(n!) < O(n^n)
